### Slide 11: GPU 并行架构
**核心原理：**
```
传统方案：
CPU ↔ GPU (数据传输瓶颈)
  
Isaac 方案：
GPU [Physics → Observation → Policy → Action]
     (全部在 GPU 上完成)
```

**性能优势：**
- 消除 CPU-GPU 数据传输
- 数万环境并行仿真
- 端到端延迟 < 1ms

**视觉建议：** 架构对比图
