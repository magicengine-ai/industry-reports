1. 执行摘要

Claude Code 是 Anthropic 官方推出的终端 AI 编程助手，通过自然语言命令帮助开发者执行代码编辑、文件操作、Git 工作流、代码审查等任务。2026 年 3 月 31 日，由于 npm 包发布时的打包错误，完整源码通过 sourcemap 文件泄露。

核心定位：Claude Code 是一个基于 Bun 运行时、React + Ink 终端 UI 的 Agentic Coding 工具，支持多 Agent 协作、MCP 协议、LSP 集成、技能系统等高级功能。

关键特性

  • 终端原生：基于 Ink (React for CLI) 的精美终端 UI
  • Agent 系统：支持 Sub-Agent 生成、团队协作、多 Agent 编排
  • 工具生态：40+ 内置工具（文件操作、Bash、搜索、MCP、LSP 等）
  • 技能系统：可复用工作流，支持自定义技能
  • IDE 桥接：VS Code / JetBrains 双向通信桥
  • 长期记忆：持久化记忆目录，支持团队记忆同步
  • 计划模式：Plan/Execute 双模式切换
  • 特性标志：基于 Bun 的死代码消除和特性开关

5. 核心模块详解

5.1 QueryEngine.ts (~46K 行)

核心 LLM API 调用引擎，负责与 Anthropic API 通信。

关键功能

  • 流式响应：SSE 协议接收实时输出
  • Tool Call 循环：处理多轮工具调用
  • Thinking 模式：支持推理模式
  • 重试逻辑：指数退避重试
  • Token 计数：实时追踪输入/输出 Token

5.2 Tool.ts (~29K 行)

定义所有工具的基础类型和接口。

核心定义

  • Input Schema: Zod 验证的输入模式
  • Permission Model: 权限模型（自动/手动/计划模式）
  • Progress State: 工具执行进度状态
  • Result Types: 结构化返回类型

5.3 commands.ts (~25K 行)

管理所有 Slash 命令的注册和执行。

命令分类

  • Git 命令：/commit, /diff, /review
  • 配置命令：/config, /mcp, /memory
  • 诊断命令：/doctor, /cost, /context
  • 模式命令：/vim, /tasks, /skills
  • 认证命令：/login, /logout

5.4 main.tsx

应用入口，Commander.js CLI 解析器 + React/Ink 渲染器初始化。

启动优化

// 并行预取（启动前执行）
startMdmRawRead()      // MDM 设置读取

apiPreconnect()         // API 预连接

// 懒加载（按需加载）
const otel = await import('@opentelemetry/sdk-node')  // ~400KB

10. 安全与权限

10.1 权限系统

每个工具调用都会检查权限，支持多种权限模式：

  • default - 默认模式，需要用户确认
  • plan - 计划模式，批量确认
  • bypassPermissions - 绕过权限（开发用）
  • auto - 自动模式，基于规则自动批准

10.2 Bash 安全

  • 破坏性命令警告：rm -rf, dd 等命令会提示确认
  • 路径验证：防止访问敏感目录
  • 只读模式验证：Plan 模式下禁止写操作
  • Sandbox 检测：决定是否使用沙箱执行

10.3 特性标志

使用 Bun 的 bun:bundle 特性标志实现死代码消除：

import { feature } from 'bun:bundle'

const voiceCommand = feature('VOICE_MODE')
  ? require('./commands/voice/index.js').default
  : null

// 未启用的代码在构建时完全移除

已知特性标志：PROACTIVE, KAIROS, BRIDGE_MODE, DAEMON, VOICE_MODE, AGENT_TRIGGERS, MONITOR_TOOL

12. 总结

12.1 架构优势

  • 终端原生体验：Ink + React 构建的精美终端 UI
  • 性能优化：Bun 运行时 + 并行预取 + 懒加载
  • 工具丰富：40+ 工具覆盖开发全流程
  • IDE 集成：Bridge 系统实现 IDE 内嵌
  • 可扩展性：技能系统、插件系统、MCP 协议
  • 安全设计：权限系统、Bash 安全校验、特性标志

12.2 技术亮点

  • QueryEngine: 46K 行核心引擎，处理 LLM 调用全流程
  • AgentTool: Sub-Agent 生成和团队协作
  • Bridge: IDE 双向通信协议
  • memdir: 持久化记忆和团队同步
  • Bun 特性标志： 构建时死代码消除

12.3 泄露影响

⚠️ 法律风险： 此源码为 Anthropic 专有财产，泄露源于 npm 打包错误。本分析仅用于教育和研究目的，不构成使用或分发建议。
  • 对 Anthropic: 暴露内部实现细节、API 设计、安全机制
  • 对社区： 学习大型 AI Agent 项目架构的宝贵机会
  • 对竞争对手： 了解 Anthropic 技术栈和产品路线图

12.4 学习价值

Claude Code 源码展示了构建生产级 AI Agent CLI 工具的最佳实践：

  • TypeScript 大型项目管理
  • React + Ink 终端 UI 设计
  • LLM Tool Calling 循环实现
  • 多 Agent 协作架构
  • IDE 集成协议设计
  • 权限和安全机制
  • 性能优化策略
总体评价： Claude Code 是一个设计精良、工程化完善的商业级 AI 编程助手。虽然因泄露事件引发关注，但其架构设计、性能优化、用户体验等方面都值得学习参考。与 DeerFlow 相比，两者定位不同：Claude Code 专注终端和 IDE 集成，DeerFlow 专注 Web 和 IM 渠道，各有优势。
