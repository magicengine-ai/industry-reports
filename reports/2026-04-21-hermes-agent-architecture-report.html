📋 执行摘要

Hermes Agent 是 Nous Research 开发的开源自托管个人 AI Agent，核心理念是"The Agent That Grows With You"——通过内置的学习循环、持久记忆和自动技能生成，实现跨会话的持续进化。

本报告深度拆解 Hermes Agent 的架构设计，包括：核心架构、代码结构、三层记忆系统、自学习循环、技能系统、子代理委派、消息网关、插件系统，以及与 OpenClaw 的对比分析。

🏗️ 第一章：核心架构概览

1.1 设计哲学

Hermes Agent 与传统 Agent 框架的本质区别在于自进化能力。大多数 Agent 只会在对话中"回忆"过去，而 Hermes 会评估已完成的任务，判断是否值得保留经验，并将成功模式转化为可复用的技能。

🧠 第三章：三层记忆架构

Hermes Agent 的记忆系统是其核心竞争力，采用三层架构：

3.1 会话上下文 (Session Context)

  • 存储：SQLite (hermes_state.py)，支持 FTS5 全文搜索
  • 功能：保存当前对话历史，提供上下文感知
  • 搜索：可通过 memory 工具搜索过去的对话

3.2 持久事实记忆 (Persistent Facts)

  • 存储：~/.hermes/memory/ 目录
  • 功能：存储关于用户的长期事实（项目、偏好、决策）
  • 更新：通过 Honcho 用户建模系统自动更新

3.3 过程性技能记忆 (Procedural Skills)

  • 存储：~/.hermes/skills/ 目录
  • 功能：从经验中提取的可复用代码技能
  • 生成：学习循环自动评估和生成

关键设计：记忆架构是缓存感知的 (cache-aware)，随着 Agent 学习更多，Token 账单不会无限增长。上下文压缩器会在对话超过阈值时自动总结中间部分。

🔄 第四章：自学习循环

Hermes Agent 的核心创新是内置学习循环，这是其他 Agent 框架所不具备的。

4.1 学习循环流程

任务完成
    ↓
评估任务价值 (是否值得保留经验？)
    ↓
    ├─ 否 → 丢弃
    │
    └─ 是 → 生成技能代码
            ↓
        保存到 ~/.hermes/skills/
            ↓
        下次遇到类似任务 → 自动加载技能
            ↓
        使用过程中持续改进技能
            ↓
        更新技能文件

4.2 关键特性

  • 自动评估：Agent 自己判断哪些经验值得保存
  • 技能生成：将成功经验转化为可复用的 Python 代码
  • 持续改进：使用过程中不断优化技能
  • 用户建模：Honcho 系统跟踪 12 层身份维度
  • 定期提醒：Agent 会自动" nudging "自己持久化知识

🤖 第六章：子代理委派

6.1 委派机制

主 Agent
    ↓
delegate_task() 调用
    ↓
    ├─ 单任务模式：提供目标 (+ 可选上下文、工具集)
    │
    └─ 批量模式：提供 tasks 数组（最多 3 个并行执行）
            ↓
        子代理继承：
        - API Key 和提供商配置
        - 凭证池（支持速率限制时密钥轮换）
            ↓
        执行完成后返回最终摘要
        （不暴露中间推理过程）

6.2 隔离选项

  • skip_context_files=True — 排除 SOUL.md, AGENTS.md 等上下文文件
  • skip_memory=True — 阻止记忆加载，隔离子代理记忆

6.3 使用原则

经验法则：当子任务需要推理、判断或多步骤问题解决时，使用 delegate_task。简单的命令执行应直接使用工具调用。

10.2 上下文压缩 (context_compressor.py)

当对话超过阈值时：
1. 保留最近的对话（完整）
2. 保留最早的对话（完整）
3. 压缩中间部分为摘要
4. 更新会话存储

10.3 工具发现 (model_tools.py)

discover_builtin_tools() 流程：
1. 扫描 ~/.hermes/skills/ 目录
2. 加载 SKILL.md 元数据
3. 注册 Python 工具函数
4. 生成工具描述供模型使用

📝 总结

Hermes Agent 代表了 AI Agent 框架的一个重要方向：从"一次性工具"到"持续进化的伙伴"。它的三层记忆架构和自学习循环使其能够随着使用变得越来越智能，这是传统 Agent 框架所不具备的核心能力。

核心优势：

  • 🧠 内置学习循环，自动从经验中学习
  • 📚 三层记忆架构，平衡性能和成本
  • 🔧 74+ 内置技能，兼容 agentskills.io 标准
  • 🌐 7+ 消息平台统一网关
  • 🤖 200+ 模型支持，无供应商锁定
  • 📈 快速迭代（v0.2→v0.8 仅 3 周）
